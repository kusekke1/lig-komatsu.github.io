本題

Gitとは？

分散型バージョン管理システムの一つ

バージョン管理とは

ファイルの変更履歴を管理することができ、
Git以外にもSVNなどが有名

バージョン管理の目的

  • 変更の差分を確認できる
  • 変更者と変更理由が明確になる

GitとGitHubの違いは？

  • GitHubはGitが使えるホスティングサービス
  • Git = GitHubではないので注意

ディレクターがGitを学習する理由

  • 最新バージョンを把握できる
  • 差分の把握ができる

最新バージョンを把握できる

納品時、確実に最新バージョンのファイルを納品することが可能になり、
先祖返りの防止につながる

差分の把握ができる

Gitは変更履歴を残してくれるので、
いつ、誰が、どんな修正をしたかを知ることができる

つまり

事故りにくい！！

Gitの操作

git clone

オンライン上のリポジトリからデータをパソコンにコピーする

コマンド


                            git clone https://github.com/lig-komatsu/lig-komatsu.github.io.git

git pull

オンライン上のリポジトリの更新をパソコンに同期させる

コマンド


                            git pull

git add

新しく追加したファイルをシステムへ認識させる

コマンド


                            git add

git commit

編集したファイルの情報をリポジトリへ記録させる

コマンド


                            git commit -a

git push

ローカルリポジトリの更新をオンライン上のリポジトリへ同期させる

コマンド


                            git push

Gitの利点

  • ローカルリポジトリがあるので、オフラインでも管理できる
  • 管理対象外ファイルの設定が楽
  • ブランチを切っての操作が簡単
  • もはや使ってない現場ないんじゃないかってくらいメジャー
  • etc...

つまり

利益しかない！！！
